> **작성자:** 비비안 (플랫폼 기획)  
> **프로젝트 기간:** 2026년 7월 (4주)  
> **문서 버전:** v0.1  
> **최종 수정:** 2026-07-01

---

## 1. 프로젝트 개요

### 1.1 한 줄 요약
인라인 스케이트 커뮤니티 멤버가 **경기·대회·행사 정보**를 한곳에서 확인하고, 팀 Slack 채널로 **매일 아침 9시 자동 요약 보고**를 받을 수 있는 플랫폼.

### 1.2 배경 & 문제
- 경기/대회 정보가 협회 홈페이지, SNS, 카페, 단톡 등에 **분산**되어 있음
- 새 일정을 놓치거나, 중복 공유로 채널이 noisy 해짐
- 운영진·기획자가 매일 수동으로 정보를 모아 공유하는 **반복 업무** 부담

### 1.3 목표 (7월 MVP)
| # | 목표 | 성공 기준 |
|---|------|-----------|
| G1 | Slack 연동 데일리 봇 | 매일 09:00 KST, 지정 채널에 크롤링 요약 메시지 1건 이상 전송 |
| G2 | 경기 정보 수집 | 최소 2개 이상 소스에서 일정/공지 데이터 수집 |
| G3 | 커뮤니티 허브 (후속) | 웹에서 수집된 일정 조회 + 수동 등록 가능 |

### 1.4 비목표 (7월 범위 밖)
- 회원 가입/결제/티켓팅
- 실시간 채팅
- 모바일 네이티브 앱
- AI 기반 일정 추천

## 3. Phase 1 — Slack API 연동 & 아침 9시 데일리 크롤링 보고 봇

> **7월 1주차 핵심 태스크**

### 3.1 기능 요구사항

#### F1. Slack 메시지 전송
- Bot Token으로 지정 Slack 채널에 메시지 post
- 메시지 포맷: Block Kit (제목, 날짜, 수집 건수, 링크 목록)
- 실패 시 에러 로그 + (선택) 관리자 DM 알림

#### F2. 스케줄 크론
- **매일 09:00 KST** 실행
- 로컬 개발: 수동 트리거 + 테스트용 cron
- 프로덕션: Vercel Cron / GitHub Actions / node-cron 중 1택

#### F3. 크롤링 파이프라인
- 대상 소스 (초기 후보, 확정 전):
  - 대한민국 인라인 관련 협회/연맹 공지 페이지
  - 주요 대회 등록 사이트 (예: inline-related RSS/공지)
- 수집 필드: `title`, `date`, `location`, `url`, `source`, `crawled_at`
- 중복 제거: URL 기준 dedup
- 변경 감지: 전일 대비 신규/변경 항목만 Slack에 강조

#### F4. 설정 & 시크릿
- `.env`로 Slack Token, Channel ID, Cron timezone 관리
- `.env.example` 제공 (실제 값 없음)
- Git에 시크릿 커밋 금지

### 3.2 비기능 요구사항
- 크롤링 1회 실행 ≤ 60초
- Slack API rate limit 준수 (429 시 exponential backoff)
- robots.txt / 이용약관 준수
- 로그: 수집 건수, 실패 URL, 실행 시각

### 3.3 Slack 메시지 예시

```
🛼 Inline Daily Report — 2026-07-01 (화)

📌 신규 3건 | 업데이트 1건

1. [대회] 2026 전국 인라인 마라톤 — 7/15, 서울
   https://example.com/event/123
2. [공지] 여름 시즌 등록 마감 안내 — 7/5
   https://example.com/notice/456
...

— inline-bot · sources: 2 · crawled at 09:00 KST
```

## 5. 기술 스택 (제안)

| 영역 | 선택 | 비고 |
|------|------|------|
| 런타임 | Node.js 20+ | Slack SDK, 크롤링 생태계 |
| 언어 | TypeScript | 타입 안정성 |
| Slack | `@slack/web-api` | Bot Token 방식 |
| 크롤링 | `cheerio` + `fetch` | 정적 HTML 우선 |
| 스케줄 | Vercel Cron 또는 GitHub Actions | 서버리스 친화 |
| 웹 (Phase 2+) | Next.js (상위 `community_app`) | 기존 프로젝트 재사용 |
